# Plan: music-support-dashboard

## Executive Summary

| 관점 | 내용 |
|------|------|
| **문제** | 음악 아티스트 지원사업 정보가 여러 사이트에 흩어져 있어, 매번 수동으로 찾아야 하는 번거로움 |
| **해결** | 국내 주요 지원사업 출처를 자동으로 수집·정리하여 한 곳에서 볼 수 있는 웹 대시보드 |
| **기능/UX 효과** | 마감일순 정렬, 지원금액 표시, 공식 링크 제공 — 주 1회 자동 업데이트 |
| **핵심 가치** | 아티스트가 지원사업 탐색에 쓰는 시간을 주 2~3시간 → 5분으로 단축 |

---

## 1. 개요

### 1.1 목적
음악 아티스트(특히 인디/DJ/프로듀서)를 위한 국내 지원사업을 자동으로 수집하고, 한눈에 볼 수 있는 웹 대시보드를 만든다.

### 1.2 배경
- 기존 작업: `indie_artist_support_2026.py` — 수동으로 데이터를 입력해 PDF 생성
- 문제점: 새 공고가 나올 때마다 직접 찾아서 파이썬 파일을 수정해야 함
- 목표: 이 과정을 자동화하고, PDF 대신 브라우저에서 바로 볼 수 있게 만들기

---

## 2. 주요 기능 (What to Build)

### 기능 1: 지원사업 데이터 수집기 (Scraper)
- 지정된 웹사이트들을 방문해서 공고 정보를 수집
- 수집 결과를 `programs.json` 파일에 저장
- 실행 방법: 터미널에서 `python scraper.py` 입력

### 기능 2: 웹 대시보드 (Dashboard)
- `programs.json`을 읽어서 브라우저에 표시
- 마감일 임박순으로 자동 정렬
- 각 카드에 표시: 사업명, 기관, 마감일, 지원금액, 공식 링크 버튼
- 파일 하나(index.html)로 구성 — 별도 서버 불필요

### 기능 3: 주간 자동 실행 (Cron)
- Mac의 스케줄러(crontab)를 이용해 매주 월요일 오전 9시에 scraper.py 자동 실행
- 실행 결과를 로그 파일에 기록

## 3. 수집 출처

| 우선순위 | 사이트 | 특징 | 수집 방식 |
|----------|--------|------|-----------|
| 1순위 | grounz.net/announcement | 음악인 전용, 이미 정리됨 | HTML 파싱 |
| 2순위 | artnuri.or.kr | 예술 전체 지원사업 검색 | HTML 파싱 |
| 3순위 | kawf.kr | 예술인 복지·지원금 | HTML 파싱 |
| 4순위 | arko.or.kr | 한국문화예술위원회 공식 | HTML 파싱 |
| 5순위 | sfac.or.kr | 서울문화재단 | HTML 파싱 |

> **주의**: 정부 사이트들은 JavaScript로 렌더링되거나 크롤링을 막는 경우가 있어,
> 수집 실패 시 수동으로 추가할 수 있는 fallback 방식도 함께 제공.

## 4. 데이터 구조

각 지원사업은 다음 형태의 JSON으로 저장:

```json
{
  "id": "ktg-band-discovery-2026",
  "name": "KT&G 상상마당 — 2026 밴드 디스커버리",
  "organization": "KT&G 상상마당",
  "deadline": "2026-04-10",
  "amount": "800만원",
  "target": "2020년 이후 데뷔, 오리지널 곡 3곡 이상 보유 뮤지션",
  "url": "https://www.sangsangmadang.com/",
  "source": "manual",
  "urgent": true,
  "updated_at": "2026-04-05"
}
```

## 5. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 언어 | Python 3 | 이미 `indie_artist_support_2026.py` 사용 중 |
| 수집 라이브러리 | requests + BeautifulSoup4 | 설치 쉽고 초보자 친화적 |
| 데이터 저장 | JSON 파일 | 데이터베이스 없이 간단하게 |
| 대시보드 | HTML + CSS + JavaScript | 서버 없이 파일만으로 실행 |
| 자동화 | crontab (Mac 내장) | 추가 설치 불필요 |

## 6. 폴더 구조

```
music-support-dashboard/
├── scraper.py          # 웹사이트에서 공고 수집
├── programs.json       # 수집된 지원사업 데이터
├── index.html          # 웹 대시보드 (브라우저에서 열기)

├── logs/
│   └── scraper.log     # 자동 실행 기록
└── docs/               # 개발 문서
```

## 8. 완료 기준 (Done When)

- [ ] `index.html`을 더블클릭하면 대시보드가 열린다
- [ ] 지원사업이 마감일 순으로 정렬되어 표시된다
- [ ] 각 사업의 공식 링크 버튼이 동작한다
- [ ] `python scraper.py`를 실행하면 `programs.json`이 업데이트된다
- [ ] 매주 월요일 오전 9시에 자동으로 수집이 실행된다

---

## 9. 제약 사항 및 주의점

1. **크롤링 제한**: 일부 정부 사이트는 자동 수집을 막음. 이 경우 수동 추가로 대체.
2. **데이터 정확도**: 자동 수집 결과는 반드시 공식 사이트에서 확인 필요.
3. **로컬 실행**: 이 대시보드는 인터넷에 공개되지 않고, 본인 컴퓨터에서만 실행됨.
